# Demos

Real workflow runs measured against the cognitive efficiency framework. Each demo produces evaluator output and feeds back into the paper.

| Demo | Domain | Status |
|---|---|---|
| [fema-pa-review](fema-pa-review/) | Public-sector regulatory document review | ✅ Modeled case study (run through evaluator) |
| [research-synthesis](research-synthesis/) | AI-assisted research writing | Protocol ready |
| agentic-coding | AI-assisted bug resolution and PR review | Incoming |
| ai-sales-research | Account research and proposal drafting | Incoming |

See also: [demo-video-script.md](demo-video-script.md) — 3–5 min script that walks the evaluator + FEMA case study.

To add your own: copy the closest existing demo folder, run the protocol, fill out the evaluator input template, and open a PR.

**Modeled vs. measured:** the FEMA case study is *modeled* (transparent assumptions, run through the real tool) — it demonstrates the method and exposes which assumptions matter. Upgrading any demo from modeled to *measured* (real timed runs, logged accept/reject decisions) is the highest-value contribution.
